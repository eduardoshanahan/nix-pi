# Architecture Decisions

One file per decision. Use a short, sequential ID and a descriptive title in the
filename.
Example: 0001-runtime-platform.md

Template

- ID: 0001
- Title: short statement
- Date: YYYY-MM-DD
- Status: proposed/accepted/superseded
- Context: brief background
- Decision: what we chose
- Rationale: why this choice
- Consequences: tradeoffs/impacts
- References: optional links/paths

Review guidance

- Revisit decisions periodically to ensure they remain compatible.
- If a decision is invalidated, add a new decision and mark the old one as
  superseded.
